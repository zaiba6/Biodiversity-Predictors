ML diagnosis
- Clear target `species_richness` exists in raw file: No
- Species column available for deriving richness: valid_name
- Need aggregation to site level first: True
- Current files contain very limited continuous environmental predictors; external raster covariates are needed.

Prioritized external variables to add
1. CRITICAL
- Variable: BIO1-BIO19 bioclim variables (temp/precip seasonality)
  Source: WorldClim v2.1 or CHELSA
  Merge: Extract raster values at each site lat/lon; if temporal matching needed use year-specific climatology or 2000-2020 normals
  Why/method: Core continuous predictors for Ridge/Lasso/Elastic Net, GAM smooths, and RF splits
- Variable: Elevation + terrain (slope, roughness)
  Source: SRTM 30m / GMTED2010
  Merge: Raster extraction by site coordinates
  Why/method: Controls biodiversity gradients, improves all model families
- Variable: Human pressure
  Source: Global Human Footprint 2009/2013 or Global Human Modification
  Merge: Raster extraction by site coordinates, optionally nearest-year version
  Why/method: Strong anthropogenic predictor for biodiversity loss/gain
2. RECOMMENDED
- Variable: Land cover class
  Source: ESA WorldCover (10m) or MODIS MCD12Q1
  Merge: Spatial join point-to-raster, then one-hot encode categories
  Why/method: Useful categorical context for RF and interaction terms
- Variable: Primary productivity (NDVI/EVI)
  Source: MODIS MOD13Q1/MOD13A2
  Merge: Year-matched seasonal or annual aggregates extracted at coordinates
  Why/method: Dynamic productivity signal for GAM/RF
- Variable: Soil variables (terrestrial)
  Source: SoilGrids (pH, SOC, texture)
  Merge: Raster extraction by lat/lon
  Why/method: Key edaphic constraints for terrestrial studies
- Variable: Ocean variables (marine studies)
  Source: Bio-ORACLE/MARSPEC/NOAA OISST
  Merge: Extract SST, salinity, chlorophyll by lat/lon; match year/month when possible
  Why/method: Essential for marine biodiversity responses
3. OPTIONAL
- Variable: Protected area status
  Source: WDPA polygons
  Merge: Point-in-polygon join
  Why/method: Policy interpretation and management relevance
- Variable: Accessibility metrics
  Source: Global roads/settlements (OSM/GRIP)
  Merge: Compute distance-to-road/city from coordinates
  Why/method: Captures disturbance and sampling bias
- Variable: Climate variability/trends
  Source: TerraClimate or ERA5
  Merge: Compute site-level interannual SD/trend using year series
  Why/method: Can improve temporal prediction
